連絡なし

らくらくんに対応開始を記録

BizBrowser を起動

  • Windows ボタンを押下
  • OpenStream→BizBrowser DT(32bit)をクリック

運用らくらくんにアクセス

  • アドレスバーに http://172.18.2.197:8081/crs/LoginScreen.crs と入力
  • 「＞」ボタンを押下

運用らくらくんにログイン

  • 「ID」欄に自分の ID を入力
  • 「password」欄に自分のパスワードを入力
  • 「ログインボタン」を押下

対応開始を記録する

対応するトランザクションに起因するアラートに対応中の印をつけます。

[!NOTE] トランザクションとアラートは 1:N の関係。
ACMS では自動リトライ機能があるため、複数のアラートが発報される可能性があります。

  • 「エラー対応画面へ」ボタンを押下

対象のアラートを検索する

以下の項目を入力してアラートを検索します。

  • 運用グループ={{ result.operation_group_code }}
  • ユーザ={{ result.user_code }}
  • ファイル={{ result.file_code }}

「検索」ボタンを押下

対象のアラートに対応開始を記録する

  • 該当レコードの「ステータス」欄をダブルクリック
  • 「対応中」と記入
  • 「担当者」欄をダブルクリックし、自分の苗字を漢字で入力
  • 入力後、「保存」ボタンを押下

送受信トランザクションを中止に状態変更する

E2X を開く

対象のトランザクションを検索する

以下の項目を入力して検索を実施します。

  • 運用グループ={{ result.operation_group_code }}
  • ユーザ={{ result.user_code }}
  • ファイル={{ result.file_code }}
  • 状態=停止

「照会」ボタンを押下

状態を中止に変更する

  • 該当するレコードを選択
  • 「操作」→「中止」を選択し、実行
  • 状態が「中止」になったことを確認

らくらくんに対応完了を記録

対応完了を記録する

アラート本文に「{{ result.error_message }}」が含まれるレコードに対し、以下を実行します。

  • 「対応内容」欄に対応履歴を入力
  • 「ステータス」欄に「完了」と記入し、「保存」を押下

リトライオーバー行の更新

  • 「ステータス」欄をダブルクリックし、「不要」と記入
  • 「保存」ボタンを押下
